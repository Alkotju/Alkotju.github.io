ЧТО ТАКОЕ ANGULARJS?

AngularJS — JavaScript-фреймворк с открытым исходным кодом. Предназначен для разработки одностраничных приложений. Его цель — расширение браузерных приложений на основе MVC-шаблона, а также упрощение тестирования и разработки.

AngularJS расширяет HTML новыми атрибутами. AngularJS идеально подходит для одностраничных приложений (SPA). AngularJS легко изучить.

ПОЧЕМУ ИМЕННО ANGULARJS?

HTML отлично подходит для объявления статических документов, но он дает сбои, когда мы пытаемся использовать его для объявления динамических представлений в веб-приложениях. AngularJS позволяет расширить словарь HTML для вашего приложения. Получившаяся среда необычайно выразительна, читабельна и быстро разрабатывается.

Альтернативы: Другие фреймворки решают недостатки HTML, либо абстрагируясь от HTML, CSS и/или JavaScript, либо предоставляя императивный способ манипулирования DOM. Ни то, ни другое не решает основную проблему - HTML не был разработан для динамических представлений.

Расширяемость: AngularJS - это набор инструментов для создания фреймворка, наиболее подходящего для разработки вашего приложения. Он полностью расширяем и хорошо работает с другими библиотеками.

ОСНОВНЫЕ ВОЗМОЖНОСТИ

Связывание данных: Привязка данных - это автоматический способ обновления представления при изменении модели, а также обновления модели при изменении представления. Это замечательно, потому что исключает манипуляции с DOM из списка того, о чем вам нужно беспокоиться.

Контроллер: Контроллеры - это поведение, стоящее за элементами DOM. AngularJS позволяет выразить это поведение в удобной для чтения форме, не прибегая к обычному шаблону обновления DOM, регистрации обратных вызовов или наблюдения за изменениями модели.

Простой JavaScript: В отличие от других фреймворков, здесь нет необходимости наследоваться от собственных типов, чтобы обернуть модель в методы-аксессоры. Модели AngularJS - это обычные объекты JavaScript. Это делает ваш код легким для тестирования, поддержки, повторного использования.

Развитие и особенности AngularJS

AngularJS — это структурный фреймворк для динамических веб-приложений, разработанный компанией Google и впервые выпущенный в 2010 году. Он позволяет использовать HTML в качестве языка шаблонов и расширяет его синтаксис для создания четко структурированных и легко тестируемых приложений. Одной из ключевых концепций AngularJS является двустороннее связывание данных, которое обеспечивает автоматическую синхронизацию между моделью и представлением, значительно упрощая разработку интерактивных пользовательских интерфейсов.

С развитием технологий и появлением новых требований к веб-приложениям, команда разработчиков приняла решение полностью переписать фреймворк, и в 2016 году был представлен Angular 2, который существенно отличается от своего предшественника. Несмотря на это, AngularJS продолжает использоваться в существующих проектах, и его концепции остаются актуальными для понимания основ построения современных одностраничных приложений.

Благодаря своей гибкости и расширяемости, AngularJS стал основой для многих крупных веб-приложений и оказал значительное влияние на развитие фронтенд-разработки в целом.

ЭВОЛЮЦИЯ ANGULAR: ОТ ANGULARJS К ANGULAR

Хотя AngularJS (Angular 1.x) больше не получает активных обновлений, экосистема Angular продолжает активно развиваться:

Angular Ivy

Ivy — новый компилятор и движок рендеринга, представленный в Angular 9, который существенно улучшает производительность:

  • Значительное уменьшение размера пакетов
  • Более быстрая компиляция и более быстрое время загрузки
  • Улучшенная отладка

Angular Signals

Signals — новый механизм реактивности в Angular 16+, который обеспечивает более простую и эффективную работу с состоянием:

import { signal, computed } from '@angular/core';

// Создание сигнала
const count = signal(0);

// Изменение значения
count.set(5);
count.update(value => value + 1);

// Вычисляемые значения
const doubleCount = computed(() => count() * 2);

Standalone Components

Новый подход к определению компонентов без необходимости объявления их в модулях:

import { Component } from '@angular/core';

@Component({
  standalone: true,
  selector: 'app-hello',
  imports: [CommonModule],
  template: `
Hello, World!
`
})
export class HelloComponent {}

Интеграция с веб-компонентами

Angular теперь лучше поддерживает создание и использование веб-компонентов:

import { Component, ViewEncapsulation } from '@angular/core';

@Component({
  selector: 'custom-button',
  template: ``,
  styles: [`.custom-btn { ... }`],
  encapsulation: ViewEncapsulation.ShadowDom
})
export class CustomButtonComponent {}

Angular DevTools

Официальное расширение для браузера, которое упрощает отладку и профилирование Angular-приложений:

  • Визуализация компонентной структуры
  • Изучение и изменение состояния компонентов в реальном времени
  • Профилирование производительности

Server-Side Rendering и Hydration

Улучшенная поддержка SSR через Angular Universal и новый механизм гидратации:

import { bootstrapApplication } from '@angular/platform-browser';
import { appConfig } from './app/app.config';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent, appConfig)
  .catch(err => console.error(err));

Особенности AngularJS

MVC архитектура

AngularJS следует шаблону проектирования MVC (Model-View-Controller), который разделяет приложение на три отдельных компонента, что делает код более организованным и легким для поддержки.

Директивы

С помощью директив AngularJS можно создавать пользовательские HTML-теги и атрибуты, чтобы добавить поведение некоторым элементам. Популярные директивы включают ng-app, ng-model, ng-bind, ng-repeat и другие.

Двустороннее связывание данных

Двустороннее связывание данных в AngularJS является наиболее примечательной особенностью: оно уменьшает количество кода, освобождая сервер от работы с шаблонами. Изменения в модели автоматически отражаются в представлении и наоборот.
